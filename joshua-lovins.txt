    1. What is Semantic HTML? 
HTML tags are semantic if they not only partition content but also describe it.
    2. What is HTML used for? 
Structuring web pages.
    3. What is an attribute and where do we put it?
Attributes are a means of distinguishing between instances of identical tags and/or abstracting from multiple different tags.

    22. What is a parent child selector? When would this be useful?
If you want to select only those instances of a given element which fall within a specified instance of another element occurring elsewhere on the document in question. That is, if you have
	<section id="1"><p>First paragraph</p></section><section id="2"><p>Second paragraph.</p></section>
you could alter the content "First paragraph" without altering the content "Second paragraph" and without having to give ids to the <p> tags of those two paragraphs by making use of the specificity provided by the id tags of the parent elements, viz., the section tags. Like this
section#1 > p {

}

    37. Walk me through Lambda's git flow.
Fork it on GitHub.
Copy the link of the forked GitHub repo.
Go to terminal and type:
	git clone https://github.com/jml1996/Git-Flow-Practice.git
To make a branch from the master version, type:
	git checkout -b "[name of branch]"
To check that you are in the correct branch (not master) on Mac, type:
	git branch
Make changes.
To check on changes, type:
	git status
Or:
	git diff
To specify that all changes are to be committed, type:
	git add .
To commit those changes, type:
	git commit -m "[message describing changes]"
To push to GitHub, type:
	git push -u origin [name of branch]
Go to GitHub.
Pull request that the branch be merged with the master (not with the original that was forked).
